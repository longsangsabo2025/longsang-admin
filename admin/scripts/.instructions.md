# Scripts — Agent Instructions

> 200+ utility scripts for migrations, testing, seeding, and setup.

## ⚠️ Warning

This folder is LARGE and contains many one-time scripts. Before creating a new script, check if one already exists:
- `scripts/run-*` — migration runners
- `scripts/test-*` — test scripts
- `scripts/seed-*` — data seeding
- `scripts/setup-*` — environment setup
- `scripts/create-*` — table/resource creation
- `scripts/check-*` — validation/diagnostics

## Script Naming Convention

```
{action}-{feature}-{detail}.{ext}

Examples:
  run-brain-migration.js
  test-academy-api.mjs
  seed-project-data.cjs
  setup-mcp-supabase.js
  check-db-schema.js
```

## File Extensions

| Extension | Module System | When to use |
|-----------|--------------|-------------|
| `.js` | Depends on package.json `"type"` | Default |
| `.mjs` | ESM (`import/export`) | Force ESM |
| `.cjs` | CommonJS (`require`) | Force CommonJS |

## Standard Script Header

```javascript
import { createClient } from '@supabase/supabase-js';
import dotenv from 'dotenv';
dotenv.config();

const supabase = createClient(
  process.env.SUPABASE_URL || process.env.VITE_SUPABASE_URL,
  process.env.SUPABASE_SERVICE_ROLE_KEY || process.env.SUPABASE_SERVICE_KEY
);
```

## Rules

1. **Check existing scripts first** — many duplicate/similar scripts exist
2. **Use `.mjs` for new scripts** — consistent ESM usage
3. **Always load dotenv** — scripts run outside Vite
4. **Log progress** — use emoji prefixes (🚀 start, ✅ success, ❌ error)
5. **Don't modify old migration scripts** — they may have already been run
